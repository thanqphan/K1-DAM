## About The Project
This project is a **Digital Asset Management (DAM)** system built using .NET Core, focusing on permission management for files and folders across multiple drives. The system allows users to manage digital assets, set permissions, and enforce access control based on roles.

## Getting Started

This section will guide you through the steps required to set up and run the project on your local machine. Follow these instructions to get a local copy up and running.


### Prerequisites

Before you begin, ensure you have the following installed on your system:

- **Visual Studio 2022 or later**: Make sure you have the latest version installed, including the **.NET desktop development** and **ASP.NET and web development** workloads.
- **SQL Server**: Ensure you have SQL Server installed and running. Alternatively, you can use a SQL Server instance in the cloud.
- **Git**: Git is required to clone the repository. If you don't have it installed, you can [Download Git](https://git-scm.com/downloads).

### Installation

Follow these steps to set up the project locally using PowerShell within Visual Studio:

1. **Clone the Repository**

   Open Visual Studio and navigate to the **Terminal** using the shortcut `Ctrl + ~` or by going to `View > Terminal`. 

   Use the following command to clone the repository:
   ```powershell
   git clone https://github.com/thanqphan/K1-DAM.git
   ```
   Navigate into the project directory:
   ```
   dotnet restore
    ```
2. **Restore Dependencies**

   Restore the project dependencies using the .NET CLI within the Visual Studio Terminal:
    ```powershell
    dotnet restore
    ```
3. **Set Up the Database**

   Open the appsettings.json file located in the src/API folder within Visual Studio. Update the connection string with your SQL Server details:
   ```
   "ConnectionStrings": {
      "DefaultConnection": "Server=your-server;Database=your-database;User Id=your-username;Password=your-password;"
   }
   ```
   Apply migrations to create the database schema:
   ```
   dotnet ef database update
   ```
4. **Run the Application**
    Run the application locally from within Visual Studio:
    ```
    dotnet run
    ```
    The API will be available by default.

 5. **Running the Tests**
    
     To ensure everything is working as expected, you can run the unit tests directly from Visual Studio Test Explorer or using the following command in the Terminal:
     ```
     dotnet test
     ```
